#Starting from Visual Studio Code

This assumes Visual Studio Code is already open and you are inside the dev folder we set up on the first day of class.

Additionally, these instructions will work with another code editor or an HTML template from the web, it is just that the default CSS and JS files will have different names.

Getting Started

  • Navigate to the folder you want to put your homework in.
    • Labs for labs
    • Exercises for Exercises
    • Code Alongs for Code alongs
    • Activities for Activities (if you set one up)
  • Right click on that folder and select Add Folder"
  • Enter the name of the project
    • e.g. "DOM-Exercise-01", "jQuery-Lab-Part-01", etc.
  • Press Enter
  • Once the folder has been created, right click on it and select New File
  • Type index.html then press Enter
  • Go into the index.html file you just created
  • Type the letters "html" and you will see a menu pop up right where you are typing. Choose second option with a square that has a dotted bottom (representing a code block) and it will insert an HTML template into the file for you
    • Either navigate to it using your arrow keys and pressing enter OR
    • Click it with your mouse

  • If you will not be using CSS or JavaScript

    • Comment out or remove completely the <script> tag that loads main.js
    • Comment out or remove completely the <link> tag that loads main.css

  • If you will be using Javascript...

    • Go into the index.html file and move the script tag that loads main.js line down to just before the closing </body> tag
    • Right click the top folder of your project and select New File again
    • Type main.js and hit Enter

  • If you will be using CSS...

    • Right click the top folder of your project and select New File again
    • Type main.css and hit Enter
  • Now, right click on the top folder of your project and select "Open in Terminal"
  • Within the terminal, type git init and hit enter
  • Open up a web browser and head to GitHub
  • Create a new repository there that has the same name as your project's top folder
  • After it has been created, copy the link under Quick setup — if you’ve done this kind of thing before
    • Note that you can just click the button at the end of the URL to copy it, you don't have to highlight it!
  • Back in Visual Studio code, type git remote add origin then paste the URL. Hit enter
    • Make sure to include the space after origin and before the URL!
  • Next, type git status and hit enter to ensure your project is ready for git
  • If you've been following these steps, the file(s) you've added should be in red. That's because they need to be added to git to be tracked
  • Do this by typing git add . and hitting enter
    • The period means "all"
  • Type git status again, and your file(s) should now be in green
  • Next, type git commit -a --m "init commit" and hit enter
    • The -a means all
    • The --m means you're going to add a message
    • The " is the start and end of your commit message
    • Your message should change after this, where you say what you've actually changed since the last time you commited
  • Type git status and hit Enter. You should get a message that there is nothing to commit, working tree clean
    • This means that all changes have been committed. If you change another file, this message will change, and that file will be listed as something that needs to be committed.
  • Now you're ready to push! Do so by typing git push then hitting Enter
  • Oops, that doesn't work. A command will show up in the terminal with a message explaining that basically, this is your first time pushing to this remote branch. You gotta do some special commands to set this up for future pushes, and to make this first push work.
  • Copy the command it suggests you, paste it, and hit enter
    • For convenience, that command is: git push --set-upstream origin master
  • Go back to github and refresh. You will see your files are now available there, and your commit message. Yay! Now you can commit and push any remaining changes
    • Do so by using the same git commit -a --m "YOUR MESSAGE HERE" command, and then git push
    • Since the next time you push won't be your first time, you don't need to add anything about upstreams to that command.
